Contexto de plataforma

CodePen es un taller y showroom visual

Separá estructura, apariencia y comportamiento como si fueran tres cajones distintos. Eso hace más fácil entender qué hace cada parte y cómo mejorarla.

En la era IA: cuando la interfaz cambia, tener un espacio simple para probar y mostrar te evita perderte en una herramienta más grande de lo necesario.

En esta ruta: CodePen aparece cuando ya tenés el resultado y querés mostrarlo claro, con el preview visible y cada panel ordenado.

3 hacks

  • Usá un panel por vez para no mezclar todo.
  • Mirá el preview temprano para detectar rápido si algo cambió.
  • Si la imagen no carga, revisá que la URL sea pública HTTPS.

Confusión común

CodePen no es “todo en un solo bloque”. Separar HTML, CSS y JS es justamente lo que ayuda a aprender sin enredarse.

Qué es CodePen

Es un editor web donde cada pieza vive separada: estructura, estilo y comportamiento. Para el estudiante, eso ayuda a ver qué hace cada cosa sin mezclar responsabilidades.

Meta

Lograr un preview visible y fácil de explicar.

Regla práctica

Copiá solo lo necesario para el panel correcto.

Qué significa que salió bien

  • El preview muestra el avatar exportado desde Colab.
  • No hay referencias a archivos locales del repositorio dentro de CodePen.
  • Los paneles se pueden editar por separado sin confusión.
  • La persona puede explicar qué cambió en HTML, CSS y JS.

Errores comunes y cómo leerlos

1

Copiar el encabezado completo

No hace falta llevar el <link> local ni el <script> local. En CodePen, eso estorba.

2

Usar una URL privada

Si la imagen no es pública HTTPS, CodePen no la puede mostrar. El problema no está en el panel, sino en el acceso al archivo.

3

Mezclar todo en un solo lugar

Si HTML, CSS y JS se mezclan, se vuelve difícil aprender qué hace cada parte. Separarlos es parte del objetivo.
