# Teacher Flow: CodePen Workshop

This guide helps facilitators review the MVP without drifting into future platform ideas.

## Review order

1. Confirm the spreadsheet → Colab → CodePen path is intact.
2. Check that the CodePen starter keeps HTML, CSS, and JS separate.
3. Verify the asset URL is public HTTPS and matches the exported GIF handoff.
4. Look for the fallback message before judging the starter as broken.

## What good looks like

| Checkpoint | Expectation |
|---|---|
| Starter structure | Learners can edit each panel independently |
| Asset handoff | A public repository-hosted URL is enough for v1 |
| Workshop scope | Small customization wins, not a full app build |
| Classroom support | The docs tell learners where to edit and where not to |

## Non-goals for MVP

- Avatar invitations or shared spaces
- Maze, route, or question-gate mechanics
- Community editing or teacher-authored platform tools
- Backend services or a generator app

## Teaching note

Keep the emphasis on the learning path, not on feature breadth. The MVP is a static kit, and that is intentional.
