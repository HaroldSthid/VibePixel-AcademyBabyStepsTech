Rol de GitHub

Repositorio como biblioteca de referencia y puerta amable para entregar evidencia.

Qué es este proyecto

Un kit público para que una clase construya avatares simples y entienda el recorrido completo: de los datos al boceto, del boceto a una imagen animada, y de esa imagen a un espacio web editable.

Para quién es

Estudiantes que están empezando y docentes que quieren enseñar tecnología con pasos visibles, no con cajas negras.

Qué se crea

Un avatar, una pequeña historia visual y un taller web donde se pueden probar accesorios, textos e interacciones.

Qué se aprende

Color, datos, validación, Python, HTML, CSS, JavaScript y, sobre todo, cómo conectar herramientas con intención.

Ruta sugerida

1

Resolvé el desafío RGB

Abrí el CSV y mirá la vista previa PNG para descubrir cómo la planilla se transforma en imagen. El éxito es reconocer el patrón de colores antes de dibujar el avatar.

2

Dibujá el avatar

Usá la planilla como cuadrícula: cada celda es una decisión visual y cada color empieza a ser dato.

3

Probá el boceto en Colab

Colab funciona como laboratorio: valida colores, muestra el resultado y exporta el avatar como imagen o GIF. El éxito es ver la vista previa y el archivo de salida.

4

Llevalo a CodePen

Pegá HTML, CSS y JS en paneles separados para ver cómo el avatar entra a un espacio web editable. El éxito es que cada panel quede claro y el preview cargue la imagen pública.

5

Abrí GitHub sin miedo

Usá GitHub para preparar tu evidencia y dejar clara tu propuesta de entrega/cambio. El éxito es que sabés dónde subir, qué escribir y qué pedir que revisen.

6

Compartí y reflexioná

Mostrá qué cambiaste, qué aprendiste y qué parte querés que revisen primero en la PR.

Los tres talleres

  • Planilla: es el cuaderno visual. Sirve para pensar el avatar como mapa, con colores Hex/RGB como se usan en proyectos reales.
  • Colab: es el laboratorio. Sirve para ensayar, equivocarse con seguridad y transformar datos en una imagen que se puede compartir.
  • CodePen: es la mesa de armado web. Sirve para convertir el avatar en un pequeño espacio editable donde estructura, estilo y comportamiento se ven por separado.

La historia que queremos abrir

Hoy empieza con un avatar individual. Más adelante, esos avatares pueden encontrarse en espacios colectivos: recorridos, preguntas para avanzar, retos pequeños o actividades entre materias. La base sigue siendo la misma: crear, explicar, compartir y mejorar.

Primer paso real

Si estás empezando, no saltes directo al código. Abrí el onboarding, entendé qué representa cada archivo y después probá la planilla. Los buenos hábitos también se aprenden desde el primer cambio guardado.
