Contexto de plataforma

GitHub es biblioteca, mesa de revisión y carpeta de evidencia

Acá guardás versiones, pedís mirada y dejás un rastro claro de lo que cambió. Pensalo como una biblioteca donde también hay un mostrador para revisar y ordenar la entrega.

En la era IA: las interfaces cambian rápido, pero sigue siendo clave saber dónde dejar evidencia, cómo pedir revisión y cómo volver a encontrarla después.

En esta ruta: GitHub no reemplaza tu trabajo; lo ordena para que la entrega tenga un lugar seguro y entendible.

3 hacks

  • Nombrá los archivos como si fueran carteles.
  • Subí solo lo que corresponde a tu entrega.
  • Escribí un mensaje corto para recordar qué cambió.

Confusión común

No todo lo que ves hay que editarlo. A veces GitHub se usa para revisar, comparar y dejar evidencia, no para complicarte más.

Por qué GitHub aparece en este desafío

Porque necesitamos una forma simple de entregar evidencia, revisar con orden y guardar la versión correcta del trabajo. No es para asustarte: es para que tu entrega tenga una puerta clara y tu facilitador pueda ayudarte sin perseguir archivos sueltos.

Primero lo mínimo

No hace falta aprender toda la historia de Git. Solo lo justo para entregar bien.

Lo que buscamos

Que puedas mostrar CSV, PNG y reflexión con una propuesta de entrega/cambio clara.

Conceptos mínimos, en palabras simples

Palabra Acá significa
Repositorio El proyecto guardado online, como una carpeta pública o compartida.
Archivo Tu evidencia: CSV, PNG, reflexión o README.
submissions/<ALIAS>/ Tu espacio de entrega dentro del proyecto.
Commit Un cambio guardado con un mensaje corto.
Pull Request Una propuesta de entrega/cambio para que alguien la revise.
Review Alguien mira si está claro, completo y seguro.
Merge La entrega fue aceptada y pasa a formar parte del proyecto.

Ruta corta usando solo GitHub web

1

Entrá al repositorio

Abrí el proyecto en GitHub. Pensalo como abrir la carpeta general donde vive la actividad.

Señal de éxito: ves el nombre del proyecto y encontrás la carpeta submissions/.

2

Buscá o creá tu alias

Si ya existe submissions/<ALIAS>/, abrilo. Si no existe, crealo agregando un archivo dentro de esa ruta, por ejemplo un README.md.

Señal de éxito: ves tu alias y sabés dónde irá tu entrega.

3

Subí la evidencia

Usá Add file → Upload files o Create new file. Sumá tu CSV, tu PNG y la reflexión breve.

Señal de éxito: los archivos aparecen en la carpeta correcta y se entiende qué aporta cada uno.

4

Escribí un mensaje corto

Guardá el cambio con una frase simple, como “Entrega RGB de HARPIR”. Ese mensaje es el recordatorio de qué cambió.

Señal de éxito: ves el cambio guardado y podés volver a encontrarlo.

5

Abrí la propuesta de entrega/cambio

Creá la Pull Request desde GitHub. Acá la llamamos propuesta de entrega/cambio: es la forma amable de pedir revisión.

Señal de éxito: la PR queda abierta con título claro y la plantilla completada.

6

Completá la plantilla y pedí revisión

Decí qué entregas, qué querés que miren primero y qué detalle puede ayudar a entender tu trabajo. Después esperá la revisión y, si hace falta, ajustá.

Señal de éxito: la persona que revisa entiende tu intención sin tener que adivinar.

Si GitHub te pide algo inesperado

  • Si te pide iniciar sesión o crear cuenta, usá la cuenta que te indicó el facilitador. Si no estás seguro, preguntá antes de usar una cuenta personal o privada.
  • Si te dice que no tenés permiso, elegí fork o create branch si aparece, o pasá a modo guiado / pedile ayuda al facilitador.
  • Si ves “create a new branch and start a pull request”, eso está bien: aceptalo para tu entrega.
  • Si se siente demasiado, no sigas solo: usá modo guiado o mandá los archivos al facilitador.

Qué revisar en cada paso

  • Repositorio abierto: encontraste el proyecto correcto.
  • Alias listo: sabés dónde vive tu entrega.
  • Archivos subidos: CSV, PNG y reflexión están juntos.
  • Commit guardado: el mensaje dice de qué trata el cambio.
  • PR abierta: la propuesta quedó visible para revisión.
  • Plantilla completa: nadie tiene que adivinar qué querías entregar.

Qué no subir

  • Datos privados.
  • Fotos personales sin permiso explícito.
  • Contraseñas o cuentas privadas.
  • Documentos sensibles o información que no querés compartir.

Si esto te pesa demasiado

No fuerces el flujo completo solo para no pedir ayuda. Mandá los archivos al facilitador o arrancá en modo guiado primero. Lo importante es entregar bien, no sufrir en silencio.
